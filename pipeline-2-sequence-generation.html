Per-Account Sequence Generation From a single named account to a validated, personalized outreach sequence per buyer

For one account at a time, this pipeline checks whether we've engaged them before, finds and qualifies the right people, researches the company, debates the strongest message for each person, writes a full multi-touch sequence, polishes the voice, and gates it for quality before it ships. It runs across two execution environments — the first two stages need live tool access only available inside a Claude Code session; the rest run unattended from the terminal.

How the runner orchestrates Stages 2–7

run-pipeline.js

The terminal runner is more than a sequencer. It discovers work, parallelizes across accounts, gates unsafe stages, resolves account names fuzzily, and writes its own audit trail. Each account's stages stay strictly sequential; only the cross-account fan-out is parallel.

💰

Cost tracking is instrumented at every API call. Every agent imports { logUsage } from lib/usage-logger.js and appends one entry per client.messages.create() response to sequences/<Account>/cost-log.json — model, tokens, cache reads/writes, USD cost. research-synthesis-agent.js and debate-agent-v2.js are not yet instrumented, which is why Stages 2 and 3 carry the cost-untracked badge above.

Accumulated Account Intelligence Layer

lib/account-knowledge.js

Every LLM-driven stage opens by building a preamble — a plain-English projection of everything currently known about the account — and prepends it to every model call. This is how a learning made in Stage 2 reaches the writer in Stage 4. The preamble has three scopes; each stage requests the one it needs.

Shared State: pipeline_context.json

one file per account · lib/pipeline-context.js

There is one state file per account at sequences/{Account}/pipeline_context.json. It is additive — each stage appends to its own section and stamps stages_completed, never overwriting other sections. Re-running a finished stage triggers an overwrite confirmation.

Inside Stage 3: The Debate Engine

debate-agent-v2.js

For each qualified persona, four AI agents argue out the strongest outreach angle. They run in ordered phases — the arguers run in true parallel across all personas, the judge runs last. The output (winning_hook + winning_angle) is the strategic backbone the writer builds on.

The 6 Right-to-Win Categories

debate angle space

Every angle must draw from one of these six categories. The diversity rule forces breadth.

⚖️ Category diversity rule: the Challenger must pick a different right-to-win category than the Advocate. The Arbitrator enforces this — if both land on the same one, it flags a diversity_violation and resolves it before declaring a winner. This guarantees the winning hook (Email 1) and the runner-up hook (Email 3) attack from genuinely different directions.

program_status drives the angle

set in Stage 1 · steers Stage 3

During qualification, the persona agent classifies where the account's subscription/membership program stands. That single field changes the strategic frame the Debate engine is allowed to pursue.

Inside Stage 6: The Validator's Named Rules

validator-agent.js

The validator runs a battery of named checks on every sequence — most are deterministic (no API call), two use Claude. Critically, content violations no longer hard-stop the pipeline: they annotate the persona's .docx and ship as written_with_warnings. Only two things actually block export.

⛔

SOURCE_PROVENANCE_VIOLATION — the real hard block. Before any sequence is validated, the agent walks all upstream JSON and the account's pipeline_context.json for any source_url on the Tier 3 blocklist or any field tagged source_tier: 3. A single hit blocks export (exit 2) until the field is re-researched from a primary source.

✓

The other hard stop is structural: if every sequence is skipped because checkpoint names don't match the debate results, that's a wiring error, not a content one — so it halts. Everything else (em dashes, exec length, weak specificity) is annotated and the pipeline continues.
